# Sprint 1 Week 3 — Features Delivered

## Accomplished

### Frontend Application
- **React + TypeScript SPA** scaffolded with Vite, served on port 5173
- **Device list page** (`/dashboard`) — displays all devices from the backend API as clickable cards
- **Device detail page** (`/dashboard/:deviceId`) — shows device info and embedded Grafana panels
- **Grafana iframe embeds** — panels auto-refresh every 5s, show last 1h of data, use light theme
- **Manual refresh** — "Refresh Panels" button forces iframe reload via React key toggle
- **URL rewriting** — `rewriteGrafanaUrl()` translates Docker-internal URLs to browser-accessible ones
- **Responsive layout** — Tailwind CSS grid adapts from 1 to 3 columns

### Infrastructure
- **Frontend Docker service** — node:20-alpine container with hot reload via volume mount
- **Named volume** for node_modules — prevents host/container dep conflicts
- **7 services** running in Docker Compose (added frontend to existing 6)

### Developer Experience
- **Hot module reload** — edit React components, see changes instantly
- **Vitest tests** — `rewriteGrafanaUrl` covered with 4 test cases
- **Type safety** — TypeScript interfaces mirror backend Pydantic schemas
- **File-based routing** — route tree auto-generated by TanStack Router plugin

## Business Value
- First user-facing UI — stakeholders can see live device data in a browser
- Grafana dashboards embedded without requiring Grafana login
- Foundation for the customer-facing product (auth + branding in Week 4)

## Gaps Remaining
- **No authentication** — all data visible to everyone (planned: Week 4)
- **No admin panel** — device/org management still done via DB seeds or API
- **No error boundaries** — React errors crash the page
- **No production build** — dev server only, no nginx/static hosting setup
- **No E2E tests** — only unit tests for utility functions
